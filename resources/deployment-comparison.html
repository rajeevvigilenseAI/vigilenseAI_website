Security Comparison

SaaS Security Model

  • Tenant Isolation: Cryptographic separation between customers
  • Data Handling: With BYODb, your data stays in your infrastructure
  • Encryption: TLS 1.3 in transit, AES-256 at rest
  • Compliance: SOC 2 Type II, GDPR, ISO 27001
  • Access: SSO, MFA, RBAC included
  • Network: VPC peering and PrivateLink available

Self-Hosted Security Model

  • Complete Control: All components run in your environment
  • Network Isolation: Define your own network boundaries
  • Custom Encryption: Use your own key management
  • Compliance: Meet any regulatory requirement
  • Access: Integrate with your identity provider
  • Air-Gap: Operate without internet connectivity

Decision Framework

Choose SaaS if you:

  • Want the fastest time to value
  • Prefer managed infrastructure
  • Don't have air-gap requirements
  • Want automatic updates and maintenance
  • Are comfortable with BYODb for data sovereignty

Choose Self-Hosted if you:

  • Have strict data residency requirements
  • Operate air-gapped environments
  • Need complete network control
  • Have dedicated infrastructure teams
  • Require custom security controls
